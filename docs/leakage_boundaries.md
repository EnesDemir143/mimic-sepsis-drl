# Leakage Boundaries — Train-Only Fit Contract

**Phase:** 03-split-manifests-and-leakage-boundaries  
**Version:** 1.0.0  
**Date:** 2026-03-28  
**Status:** Active — binding for all downstream phases

---

## Purpose

This document defines the canonical leakage boundary for the MIMIC Sepsis
Offline RL pipeline.  Every learned transform, discretization rule, or
statistical estimate that flows into the RL dataset **must respect** the
boundary described here.

Violation of this contract invalidates offline policy evaluation results and
undermines the reproducibility claims of the project.

---

## The Core Rule

> **Fit on train. Transform everywhere.**

Any operation that *learns* a parameter from data must be applied exclusively
to the training partition.  The learned parameter is then *applied* (not
re-learned) to the validation and test partitions.

This includes, but is not limited to:

| Transform | Fit on | Apply to |
|-----------|--------|----------|
| Scaler (StandardScaler, MinMaxScaler, …) | train | train · val · test |
| Imputer (median, mean, KNN, …) | train | train · val · test |
| Outlier clip bounds (IQR, percentile, …) | train | train · val · test |
| Action bin thresholds (vasopressor / IV fluid quantiles) | train | train · val · test |
| Behaviour cloning policy (BC estimator) | train | — |
| Value function / Q-function estimates (FQE) | train | train (rollout only) |
| Any normalisation constant | train | train · val · test |

## Split Manifest as the Boundary Authority

The patient-level split manifests generated by `src/mimic_sepsis_rl/data/splits.py`
are the **single source of truth** for which patients belong to each partition.

```
data/splits/
├── train_manifest.parquet       # subject_id + stay_ids for training patients
├── validation_manifest.parquet  # subject_id + stay_ids for validation patients
├── test_manifest.parquet        # subject_id + stay_ids for test patients
└── split_summary.json           # seed, source_episode_set, per-split counts
```

Every downstream phase **must** load the manifest before processing and must not
construct its own ad-hoc splits.

### Loading the manifest

```python
from mimic_sepsis_rl.data.splits import load_manifest_parquet
from pathlib import Path

manifest = load_manifest_parquet(Path("data/splits"))
train_stays = {
    stay_id
    for r in manifest.records
    if r.split.value == "train"
    for stay_id in r.episode_keys
}
```

## Patient-Level Partitioning

Splits operate on unique `subject_id` values (MIMIC-IV patient identifiers),
not on `stay_id` or row counts.

**Why this matters:** A patient may have multiple ICU stays.  Assigning stays
at random — without anchoring to the patient — can place the same patient in
both the training and validation partitions, creating direct information leakage.

The manifest guarantees:

```
train_ids ∩ validation_ids = ∅
train_ids ∩ test_ids       = ∅
validation_ids ∩ test_ids  = ∅
```

This is enforced at generation time by `SplitManifest.has_leakage()` and by
the regression tests in `tests/data/test_split_manifests.py`.

## Seed and Reproducibility

The random seed used to generate the split is recorded in:
- `data/splits/split_summary.json` → `"seed"` field
- `SplitManifest.seed` attribute

Do not change the seed after the first split is generated and committed.
Changing the seed invalidates all downstream cached artifacts.

Default seed: **42** (from `configs/splits/default.yaml`).

## Phase-by-Phase Application

### Phase 4 — Feature Preprocessing

- Load `train_manifest.parquet` and fit all scalers, imputers, and clip bounds
  on train rows only.
- Persist the fitted transformers (e.g. as `joblib` or `pickle` artifacts).
- Apply the same fitted transformers to validation and test rows.

### Phase 5 — Action Discretization

- Learn vasopressor and IV fluid bin thresholds from train rows only.
- Persist bin edges as part of the experiment config.
- Apply the same bin edges to all partitions.

### Phase 6 — Transition Dataset Assembly

- Assemble `(s, a, r, s')` tuples using the manifest to filter rows
  correctly before writing train / val / test dataset files.

### Phase 7 & 8 — RL Training (CQL / BCQ / IQL)

- Train exclusively on the train transition dataset.
- Use validation for hyperparameter selection (early stopping, etc.).
- Touch test data only once, for final OPE reporting.

### Phase 9 — Offline Policy Evaluation (WIS, FQE)

- Evaluation metrics are reported on the test partition.
- No parameter is fitted on test data at any point.

## Automated Enforcement

The primary leakage guards are enforced by:

```
tests/data/test_split_manifests.py
```

Key test classes:
- `TestNoPatientLeakageAcrossSplits` — disjointness of the three partition sets
- `TestManifestProvenanceContract` — seed and source episode set are recorded
- `TestDeterminism` — same inputs always produce the same partition

Run with:

```bash
pytest -q tests/data/test_split_manifests.py
```

All tests must pass before any downstream phase begins processing.

## Quick Reference Checklist for Future Phases

Before implementing any learned transform in a downstream phase:

- [ ] Load the split manifest from `data/splits/`.
- [ ] Filter to train rows **before** fitting the transform.
- [ ] Persist the fitted transform as a named artifact.
- [ ] Apply the transform to val/test using the persisted artifact.
- [ ] Do **not** re-fit on val or test data.
- [ ] Add a regression test asserting the transform artifact path exists and is
      consistent with the train manifest seed.
